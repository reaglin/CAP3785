Data Quality Assessment: Identifying and Classifying Data Quality Issues

Module: Data Quality Assessment

Assignment Type: Analysis of Data

Estimated Time: Less than one hour

Submission Format: As directed by your instructor or the course learning management system

Purpose

In data-driven roles — whether in business, healthcare, science, or public policy — decisions are only as good as the data behind them. Poor-quality data leads to flawed analyses, costly mistakes, and loss of trust. This assignment gives you direct, hands-on experience recognizing and categorizing real data quality problems using four widely accepted quality dimensions: accuracy, completeness, consistency, and reliability.

By working through a realistic dataset that contains deliberate errors, you will practice the same critical evaluation skills that data analysts, data stewards, and quality assurance professionals use every day. This task directly supports your ability to meet the following course learning outcome:

  • MO1 [Analyze]: Classify a dataset's quality issues according to the dimensions of accuracy, completeness, consistency, and reliability.

Learning Outcomes

After completing this assignment, you will be able to:

  1. Analyze a provided dataset to locate specific cells or records that contain data quality problems.
  2. Classify each identified issue according to the appropriate data quality dimension (accuracy, completeness, consistency, or reliability).
  3. Justify the corrective action you would take for each issue, explaining why that action is appropriate given the dimension involved.

Key Terms and Definitions

Before you begin, review the following definitions. You will use these terms precisely in your analysis.

Accuracy
The degree to which data values correctly represent the real-world facts they are intended to describe. Example: a customer's listed age of 187 years is inaccurate because it is not a plausible real-world value.
Completeness
The degree to which all required data values are present and not missing. Example: a record with no email address in a field that is required for communications is incomplete.
Consistency
The degree to which the same data values are represented in the same format and without contradiction across the dataset. Example: a date field that contains "03/15/2023" in some rows and "15-Mar-23" in others is inconsistent.
Reliability
The degree to which data values can be trusted to be stable, repeatable, and sourced from a credible process. Example: a sensor reading that randomly doubles or halves without any real-world change in conditions is unreliable. In datasets, this often appears as duplicate records with conflicting values, or values that clearly contradict other known, verifiable data points in the same record.
Corrective Action
The specific step taken to resolve a data quality issue. Common actions include: flag for review (mark the record so a subject-matter expert can verify it), impute (fill in a missing value using a calculated or logical estimate), standardize (reformat to a consistent pattern), delete/remove (remove a duplicate or irreparably corrupt record), or correct (replace an incorrect value with the verified correct value).
