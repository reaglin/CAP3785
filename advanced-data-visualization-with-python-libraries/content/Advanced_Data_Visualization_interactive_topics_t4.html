Customization and Publication-Quality Styling

▶

Producing a visualization that communicates clearly in a journal article, conference poster, or technical report demands far more than simply calling a plot function and accepting its defaults. Default Matplotlib figures are functional but deliberately spartan; their font sizes are optimized for an interactive notebook screen, their colors are chosen for general-purpose readability, and their spacing is designed to accommodate a wide variety of content rather than your specific data. Publication-quality styling is the discipline of deliberately overriding every one of those defaults so that the finished figure looks intentional, professional, and appropriate for its target medium. This topic covers the full pipeline: applying coherent visual themes, selecting accessible color palettes, controlling typography, adding annotations and reference lines, managing layout and spacing, configuring legends and colorbars, and finally exporting the figure in a format and resolution that meets professional requirements.

Applying Themes and Style Sheets

A Matplotlib style sheet is a plain-text file that sets dozens of rcParams values simultaneously, giving every subsequent plot a coherent look without repeating the same configuration code. Calling plt.style.use('seaborn-v0_8-whitegrid') before any plotting command applies that style globally for the entire Python session. Matplotlib ships with a rich library of built-in styles — 'ggplot', 'bmh', 'fivethirtyeight', 'grayscale', and many Seaborn-derived variants — each tuned for a different aesthetic goal.

import matplotlib.pyplot as plt
import numpy as np

plt.style.use('seaborn-v0_8-whitegrid')

x = np.linspace(0, 2 * np.pi, 200)
fig, ax = plt.subplots(figsize=(6, 3))
ax.plot(x, np.sin(x), label='sin(x)')
ax.plot(x, np.cos(x), label='cos(x)')
ax.legend()
plt.tight_layout()
plt.show()

Seaborn provides its own layered theming system through sns.set_theme(). The style parameter controls background and grid elements ('white', 'whitegrid', 'dark', 'darkgrid', 'ticks'), while the context parameter scales all text and line elements for the intended viewing size. Use context='paper' for a small figure destined for a two-column journal layout, context='notebook' for exploratory work, and context='talk' or 'poster' when the audience will read from a distance.

import seaborn as sns

sns.set_theme(style='whitegrid', context='paper', font_scale=1.2)

When you want a style to apply only to a specific block of code without contaminating other figures produced later in the same script, wrap that block in the context manager plt.style.context(). Everything inside the with block uses the requested style; everything outside is unaffected.

with plt.style.context('grayscale'):
    fig, ax = plt.subplots()
    ax.plot([1, 2, 3], [4, 1, 3])
    plt.savefig('grayscale_preview.pdf', bbox_inches='tight')
# Outside the block, the previous style is restored automatically

For a research group or organization that wants a consistent house style across dozens of notebooks, the most scalable solution is a custom .mplstyle file. Place a file such as myjournal.mplstyle in Matplotlib's configuration directory (found via matplotlib.get_configdir()) or pass its full path to plt.style.use(). The file uses the same key-value syntax as rcParams:

# myjournal.mplstyle
figure.figsize    : 3.5, 2.5
font.family       : serif
font.size         : 8
axes.linewidth    : 0.8
axes.spines.top   : False
axes.spines.right : False
lines.linewidth   : 1.2

Loading this file at the top of every notebook guarantees that every figure produced by every team member will share the same line weights, font sizes, and spine configuration without manual repetition.

Selecting and Managing Color Palettes

Color is one of the most powerful and most frequently misused visual channels in data graphics. Choosing a palette is not merely an aesthetic decision; it directly affects whether readers can extract accurate information from the figure. Seaborn's sns.color_palette() function is the primary tool for inspecting and constructing palettes. Called with no arguments it returns the current palette; called with a name and an integer it returns that many colors from the named palette as a list of RGB tuples that can be passed anywhere Matplotlib accepts a color list.

palette = sns.color_palette('Set2', 6)
sns.palplot(palette)   # renders swatches in a Jupyter notebook

To apply a palette globally so that every subsequent plot automatically cycles through those colors, use sns.set_palette():

sns.set_palette('colorblind')

The choice of palette type should follow the nature of the data being encoded:

  • Sequential palettes such as 'Blues', 'viridis', 'plasma', or 'YlOrRd' progress from a light, low-saturation color to a dark, high-saturation color. They are appropriate when the variable being colored has a natural order and a meaningful zero or minimum — for example, population density, temperature, or probability.
  • Diverging palettes such as 'RdBu', 'coolwarm', or 'PuOr' have a pale, neutral midpoint flanked by two contrasting hues. They are the right choice when the data has a meaningful center (often zero) and deviations in both directions are equally important — for example, correlation coefficients, anomalies relative to a baseline, or z-scores.
  • Qualitative palettes such as 'Set1', 'Set2', 'tab10', or 'Paired' assign visually distinct hues to categorical groups without implying any ordering. They are appropriate for nominal variables like species, country, or treatment arm.

A critical professional requirement is accessibility. Roughly 8 percent of men have some form of red-green color vision deficiency, meaning a figure that relies on distinguishing red from green will be unreadable for a substantial fraction of readers. Seaborn's 'colorblind' palette is specifically designed to remain distinguishable under the most common forms of color vision deficiency. Additionally, any figure that may be printed in a journal that uses black-and-white printing for some figures must also be interpretable in grayscale. The 'viridis' and 'cividis' sequential palettes were designed to be perceptually uniform and fully legible in grayscale. For categorical line plots, combining color differences with distinct line styles (linestyle='-', '--', ':') or markers provides a redundant encoding that survives both grayscale conversion and color vision deficiency.

Customizing Fonts and Text Elements

Typography in a figure must be consistent with the typography of the surrounding document. A journal article typeset in a 10 pt serif font looks incongruous if its embedded figures use a different sans-serif font at a mismatched size. Matplotlib exposes all font properties through the rcParams dictionary, which can be modified globally before creating any figures:

import matplotlib as mpl

mpl.rcParams['font.family'] = 'serif'
mpl.rcParams['font.serif'] = ['Times New Roman', 'DejaVu Serif']
mpl.rcParams['font.size'] = 9
mpl.rcParams['axes.titlesize'] = 10
mpl.rcParams['axes.labelsize'] = 9
mpl.rcParams['xtick.labelsize'] = 8
mpl.rcParams['ytick.labelsize'] = 8
mpl.rcParams['legend.fontsize'] = 8

Individual text elements can override the global settings through keyword arguments. The most commonly used are fontsize, fontweight ('normal', 'bold'), fontstyle ('italic'), and fontfamily:

ax.set_title('Experimental Results', fontsize=11, fontweight='bold')
ax.set_xlabel('Time (s)', fontsize=9, fontfamily='sans-serif')

For figures that include mathematical notation — axis labels like R², units like μm, or equations in annotations — Matplotlib supports two rendering paths. The first is its own internal MathText engine, which interprets LaTeX-like syntax wrapped in dollar signs without requiring a LaTeX installation:

ax.set_ylabel(r'$\Delta F / F_0$ (%)')
ax.set_title(r'Mean $\pm$ SEM, $n = 12$')

The second is full external LaTeX rendering, which produces output indistinguishable from the rest of a LaTeX-typeset document. It requires a working LaTeX installation and is activated by setting plt.rcParams['text.usetex'] = True. When this is enabled, all text strings are passed through LaTeX, so ordinary text must be wrapped in \text{} commands and special characters must be escaped.

mpl.rcParams['text.usetex'] = True
mpl.rcParams['font.family'] = 'serif'
ax.set_xlabel(r'\textit{Concentration} ($\mu$mol\,L$^{-1}$)')

A frequent mistake when preparing figures for publication is choosing font sizes that look fine on a large monitor but become illegible after the figure is scaled to column width. A typical single-column figure in a two-column journal is approximately 3.5 inches wide. If the figure was designed at 7 inches and then reduced by half, all font sizes are effectively halved. The safest approach is to set figsize at the outset to match the final printed dimensions and choose font sizes of 7–9 pt, which are standard in journal figures.

Adding Annotations and Reference Lines

Annotations transform a graphic from a raw display of data into a guided narrative. They direct the reader's attention to the features that matter most and provide the contextual information needed to interpret those features correctly.

The most expressive annotation tool is ax.annotate(), which combines a text label with an optional arrow. The xy parameter specifies the point being annotated (the arrowhead destination) in data coordinates; xytext specifies the position of the text in data or axes coordinates; and arrowprops is a dictionary controlling the arrow style:

ax.annotate(
    'Local maximum',
    xy=(np.pi / 2, 1.0),          # point being annotated
    xytext=(np.pi / 2 + 0.5, 0.7),  # text position
    fontsize=8,
    arrowprops=dict(arrowstyle='->', color='black', lw=0.8),
    ha='left'
)

The arrowstyle string supports many styles: '->', '-|>', 'fancy', 'wedge', and others. The xycoords and textcoords parameters allow mixing coordinate systems; for example, placing the text at a fixed position in axes fraction coordinates (where (0, 0) is the lower-left corner and (1, 1) is the upper-right corner) while the arrow points to a data coordinate ensures the label stays in a readable position regardless of axis limits:

ax.annotate(
    'Threshold exceeded',
    xy=(3.2, 0.95),
    xycoords='data',
    xytext=(0.65, 0.92),
    textcoords='axes fraction',
    arrowprops=dict(arrowstyle='->', connectionstyle='arc3,rad=0.2')
)

For simple text labels without arrows, ax.text() is more concise. It accepts x, y, and the string, plus all the usual font keyword arguments:

ax.text(0.05, 0.95, 'Panel A', transform=ax.transAxes,
        fontsize=10, fontweight='bold', va='top')

Passing transform=ax.transAxes switches to axes fraction coordinates, which is the standard way to place panel labels like A, B, C in a consistent position across all subplots regardless of the data range.

Horizontal and vertical reference lines are drawn with ax.axhline() and ax.axvline(). They span the full width or height of the axes by default, or can be limited to a fractional range with the xmin/xmax or ymin/ymax parameters:

ax.axhline(y=0, color='black', linewidth=0.8, linestyle='--')
ax.axvline(x=treatment_start, color='gray', linewidth=1.0,
           linestyle=':', label='Treatment start')

To shade a region rather than draw a single line, use ax.axhspan() or ax.axvspan(). These functions fill a horizontal or vertical band across the axes and are ideal for marking confidence intervals, acceptable operating ranges, or periods of interest in a time series:

ax.axhspan(ymin=lower_ci, ymax=upper_ci, alpha=0.15,
           color='steelblue', label='95% CI')
ax.axvspan(xmin=phase2_start, xmax=phase2_end, alpha=0.10,
           color='orange', label='Phase II')

Fine-Tuning Layout and Spacing

Poor layout is one of the most common reasons an otherwise excellent figure looks unprofessional. Tick labels that are clipped by the figure boundary, subplots whose titles overlap with the axes below, or unequal whitespace on different sides of the figure all signal that the author did not control their output carefully.

The first decision is figure size. Setting it explicitly via plt.figure(figsize=(width, height)) or the equivalent fig, ax = plt.subplots(figsize=(width, height)) is mandatory for publication work. Dimensions should be specified in inches to match journal column widths. Common targets are 3.5 inches (single column), 5.0–5.5 inches (1.5 columns), and 7.0–7.5 inches (double column).

Once figure size is set, spacing between subplots and between plot elements and the figure boundary must be managed. plt.tight_layout() is a quick automatic solution that inspects the bounding boxes of all text and axis elements and adjusts subplot parameters to prevent overlap. Passing constrained_layout=True as a parameter to plt.subplots() activates a more robust algorithm that handles colorbars and other axes-level decorations that tight_layout() sometimes misses:

fig, axes = plt.subplots(2, 3, figsize=(7, 4.5), constrained_layout=True)

When automatic adjustment is insufficient, plt.subplots_adjust() provides manual control over six parameters:

  • left, right, top, bottom: the position of the subplot area boundaries as fractions of the figure width or height (0 to 1).
  • hspace: the fraction of the average subplot height reserved for vertical space between rows.
  • wspace: the fraction of the average subplot width reserved for horizontal space between columns.
plt.subplots_adjust(left=0.10, right=0.95, top=0.92, bottom=0.12,
                    hspace=0.40, wspace=0.30)

For multi-panel figures where different panels should occupy different amounts of space — for example, a large main plot with a narrow marginal distribution — matplotlib.gridspec.GridSpec allows precise proportional sizing and subplot spanning:

import matplotlib.gridspec as gridspec

fig = plt.figure(figsize=(7, 5))
gs = gridspec.GridSpec(2, 3, figure=fig,
                       height_ratios=[3, 1],
                       width_ratios=[2, 2, 1],
                       hspace=0.35, wspace=0.30)

ax_main   = fig.add_subplot(gs[0, :2])   # spans first two columns, first row
ax_right  = fig.add_subplot(gs[0, 2])    # third column, first row
ax_bottom = fig.add_subplot(gs[1, :])    # full width, second row

This approach gives complete control over the relative sizes of panels and is standard practice for complex figures in high-impact publications.

Configuring Legends and Colorbars

A legend must be readable and must not obscure data. Matplotlib's automatic placement algorithm (loc='best') attempts to find the emptiest region of the axes, which works well for simple figures but fails when data fills the entire axes area. In those cases, the safest strategy is to move the legend outside the plot area using bbox_to_anchor:

ax.legend(loc='upper left', bbox_to_anchor=(1.02, 1.0),
          borderaxespad=0, frameon=True, fontsize=8)
fig.savefig('figure.pdf', bbox_inches='tight')
# bbox_inches='tight' ensures the legend is not clipped at export

Additional legend customization options worth knowing:

  • ncol=2 arranges legend entries in two columns, reducing height for wide figures.
  • framealpha=0.9 controls the opacity of the legend background box.
  • handlelength=1.5 and handleheight=1.0 control the size of the color/line/marker sample in each legend entry.
  • Passing explicit handles and labels lists to ax.legend() allows complete control over which series appear and what labels they carry, enabling removal of redundant entries that are already obvious from axis labels or the figure caption.
# Remove a specific series from the legend
handles, labels = ax.get_legend_handles_labels()
filtered = [(h, l) for h, l in zip(handles, labels) if l != '_nolegend_']
ax.legend(*zip(*filtered), fontsize=8)

Colorbars appear alongside mappable plots such as heatmaps, scatter plots with a color-mapped variable, or filled contour plots. The standard call is:

scatter = ax.scatter(x, y, c=z, cmap='viridis', vmin=0, vmax=100)
cbar = plt.colorbar(scatter, ax=ax, shrink=0.85, pad=0.03)
cbar.set_label('Fluorescence Intensity (a.u.)', fontsize=8)
cbar.ax.tick_params(labelsize=7)

The shrink parameter scales the colorbar length relative to the axes height; pad controls the gap between the axes and the colorbar. For multi-axes figures, a single colorbar can serve several subplots by passing a list of axes to the ax parameter:

cbar = fig.colorbar(mappable, ax=[ax1, ax2], location='right', shrink=0.6)

Always label the colorbar with the variable name and its units. A colorbar strip with no label forces readers to consult the caption to understand what the color channel encodes, which slows comprehension.

Exporting Publication-Ready Figures

The final step is saving the figure in a format and at a resolution appropriate for its destination. The key function is plt.savefig() (or equivalently fig.savefig()), and several parameters must be set deliberately:

  • dpi: dots per inch. For raster formats, use at least 300 DPI for print; many journals require 600 DPI for line art. For vector formats, dpi controls only the resolution of any embedded raster elements (such as images used as backgrounds) and does not affect the sharpness of vector elements.
  • bbox_inches='tight': trims the saved figure to the smallest bounding box that contains all visible elements, eliminating excess white space. This parameter is almost always desirable and is especially important when legends or axis labels extend beyond the default figure boundary.
  • transparent=True: sets the figure and axes backgrounds to transparent rather than white. Use this when embedding figures in presentations with colored slide backgrounds or in documents with tinted page backgrounds.
  • format: inferred from the file extension but can be specified explicitly with format='pdf' to avoid ambiguity.
fig.savefig('figure_1.pdf', dpi=300, bbox_inches='tight')
fig.savefig('figure_1.tiff', dpi=600, bbox_inches='tight')
fig.savefig('figure_1.png', dpi=300, bbox_inches='tight', transparent=True)

The choice between vector and raster formats has significant practical consequences:

Format Type Best for Editable after export?
PDF Vector LaTeX inclusion, journal submission, infinite rescaling Yes (Illustrator, Inkscape)
SVG Vector Web publication, further editing in Inkscape Yes
EPS Vector Older journal submission systems that require EPS Yes
PNG Raster Presentations, web, journals requiring raster at 300 DPI No
TIFF Raster High-resolution journal submissions, archival storage No

Vector formats preserve every line, curve, and text character as a mathematical description, so the figure remains perfectly sharp at any zoom level and can be opened in a vector editor to adjust individual elements. Raster formats capture the figure as a fixed grid of pixels; once saved, resolution cannot be increased without resampling artifacts. As a practical rule, save a vector copy as the archival master and generate raster versions from it as needed, rather than the reverse.

One final consideration is font embedding. When saving to PDF, Matplotlib embeds font subsets by default, which is correct behavior for submission. However, if the target journal or print house requires all fonts to be outlined (converted to vector paths) to avoid font substitution, this can be achieved by opening the PDF in Inkscape or Illustrator and converting text to outlines before final submission. Alternatively, using text.usetex=True with a standard LaTeX font ensures maximum compatibility with LaTeX-based journal production workflows.
