Introduction to Matplotlib and Seaborn

▶

Data visualization is one of the most powerful tools available to analysts, scientists, and engineers working in Python. Rather than presenting raw numbers in tables, visualization transforms data into patterns, trends, and relationships that the human eye can grasp instantly. The Python ecosystem offers many libraries for creating charts and graphs, but two dominate the landscape for nearly all use cases: Matplotlib and Seaborn. Understanding what each library does, how they relate to one another, and how to configure your environment correctly is the essential first step before any serious visualization work begins.

These two libraries are not rivals — they are complementary tools designed to work together. Matplotlib provides the foundational engine, while Seaborn provides a high-level interface built on top of it. A professional Python developer fluent in both can produce everything from a quick exploratory scatter plot to a polished, publication-ready multi-panel figure.

What is Matplotlib?

Matplotlib is the oldest and most widely used plotting library in the Python ecosystem. It was created by John D. Hunter in 2003 with the explicit goal of replicating the plotting capabilities of MATLAB in Python, and it has since grown into a comprehensive, production-grade visualization toolkit. Its fundamental design philosophy is to give the programmer complete, granular control over every visual element in a figure — from the thickness of a single axis line to the exact pixel position of a legend.

At its core, Matplotlib provides a low-level, highly customizable API. "Low-level" here does not mean primitive or limited — it means that the library exposes every individual component of a chart as an object that can be independently inspected and modified. Want a dashed red border only on the top and right sides of the plot? Want tick labels rotated at 45 degrees with a specific font family and size? Matplotlib makes all of this possible because it represents every visual artifact — lines, patches, text, ticks, legends — as a Python object with configurable properties.

The most commonly used part of Matplotlib is the pyplot module, imported conventionally as plt. The pyplot module offers a MATLAB-style, stateful interface that tracks the "current figure" and "current axes" automatically, so simple plots can be created with very few lines of code:

import matplotlib.pyplot as plt

plt.plot([1, 2, 3, 4], [10, 20, 15, 25])
plt.xlabel("X Axis")
plt.ylabel("Y Axis")
plt.title("A Simple Line Plot")
plt.show()

This pyplot interface is intuitive for quick tasks, but for more complex figures, Matplotlib's object-oriented (OO) API is preferred because it makes the code explicit and avoids confusion about which figure or axes is currently "active."

Internally, Matplotlib organizes every figure into a hierarchy of objects. At the top sits the Figure object — the overall canvas. Inside the Figure live one or more Axes objects — these are the actual plot areas where data is drawn, including their own axis lines, tick marks, and labels. Everything drawn within an Axes — lines, markers, text, fills — is an Artist object. This three-tier hierarchy (Figure → Axes → Artists) is central to understanding how Matplotlib works and why it is so flexible. Every customization ultimately comes down to finding the right object in this hierarchy and setting its properties.

Because of this granular control, Matplotlib is the preferred choice whenever precise customization is required — for instance, when preparing figures for academic journals with strict formatting requirements, or when building custom chart types that no higher-level library supports out of the box.

What is Seaborn?

Seaborn was created by Michael Waskom and released in 2012. Its explicit purpose is to make statistical data visualization easier and more visually appealing by abstracting away the repetitive boilerplate that Matplotlib requires for common tasks. Where Matplotlib asks you to manually compute histograms, bin data, and draw bar patches, Seaborn does all of that internally and exposes a single function call.

Consider plotting a distribution with Matplotlib versus Seaborn:

# Matplotlib approach — manual steps
import matplotlib.pyplot as plt
import numpy as np

data = np.random.randn(500)
plt.hist(data, bins=30, edgecolor='black', color='steelblue')
plt.xlabel("Value")
plt.ylabel("Frequency")
plt.show()

# Seaborn approach — one function
import seaborn as sns
sns.histplot(data, bins=30, kde=True)
plt.show()

The Seaborn version not only requires less code but also automatically adds a kernel density estimate (KDE) curve, applies a more aesthetically refined style, and handles the color intelligently. This illustrates Seaborn's core value proposition: complex chart types with minimal code.

Seaborn excels particularly at visualizing statistical relationships, distributions, and categorical data. Its function library includes tools for pair plots (showing pairwise relationships across all columns of a dataset), heatmaps (for correlation matrices or pivot tables), box plots, violin plots, regression plots, and more. These would each require many lines of Matplotlib code to produce from scratch, but Seaborn handles them in one or two lines:

import seaborn as sns
import matplotlib.pyplot as plt

df = sns.load_dataset("penguins")

# A pair plot across all numeric columns, colored by species
sns.pairplot(df, hue="species")
plt.show()

Beyond convenience, Seaborn ships with built-in themes and color palettes designed by professionals with expertise in perceptual color science. The default Seaborn output looks polished and publication-ready without any manual styling work. This is a significant advantage for users who want good-looking charts quickly, without spending time on aesthetics.

Crucially, Seaborn's high-level interface does not eliminate flexibility. Because every Seaborn plot is ultimately a Matplotlib figure under the hood, you can continue customizing any Seaborn plot using Matplotlib commands after the Seaborn function call. This makes Seaborn a "best of both worlds" solution for most data science workflows.

The Relationship Between Matplotlib and Seaborn

Understanding the architectural relationship between these two libraries is essential for using them together effectively. The relationship is straightforward: Seaborn is built on top of Matplotlib. Every Seaborn function, internally, creates and manipulates Matplotlib Figure and Axes objects. When you call sns.histplot(), Seaborn computes the binning internally, then calls Matplotlib's drawing primitives to render rectangles, lines, and labels on a Matplotlib Axes.

This means that both libraries can — and routinely should — be used together in the same script. A common pattern is to use Seaborn to draw the main content of a chart quickly, then use Matplotlib to fine-tune the details:

import seaborn as sns
import matplotlib.pyplot as plt

df = sns.load_dataset("tips")

# Seaborn draws the main chart
ax = sns.boxplot(data=df, x="day", y="total_bill", palette="Set2")

# Matplotlib refines the details
ax.set_title("Total Bill by Day of Week", fontsize=16, fontweight='bold')
ax.set_xlabel("Day", fontsize=13)
ax.set_ylabel("Total Bill (USD)", fontsize=13)
ax.tick_params(axis='x', rotation=15)
plt.tight_layout()
plt.show()

Another important behavior to understand is that importing Seaborn automatically applies its default theme to all subsequent Matplotlib plots in the same Python session. This is because Seaborn modifies Matplotlib's global rcParams (rendering configuration parameters) when it is imported. This is usually desirable — Seaborn's defaults look better than Matplotlib's raw defaults — but it is worth knowing so you are not surprised when a bare plt.plot() call suddenly looks different after adding import seaborn as sns to your script.

This layered architecture — Matplotlib as the rendering engine, Seaborn as the high-level interface — gives Python users a uniquely powerful combination. You can write concise, expressive Seaborn code for 90% of your chart, then drop down to Matplotlib's granular API for the remaining 10% that requires precise control.

Setting Up Your Environment

Before any plotting can begin, the correct libraries must be installed and imported. The installation is straightforward using either pip or conda (for Anaconda/Miniconda environments):

# Using pip (standard Python package manager)
pip install matplotlib seaborn numpy pandas

# Using conda (Anaconda/Miniconda)
conda install matplotlib seaborn numpy pandas

It is strongly recommended to install NumPy and pandas alongside Matplotlib and Seaborn, because nearly every real visualization workflow depends on them. NumPy provides efficient array operations needed for generating and transforming numerical data, while pandas provides the DataFrame structure that Seaborn is especially optimized to work with. Most Seaborn functions accept a pandas DataFrame and a column name directly, making the data-to-chart pipeline seamless.

At the top of any visualization script, the standard imports are:

import matplotlib.pyplot as plt
import seaborn as sns
import pandas as pd
import numpy as np

These aliases — plt, sns, pd, np — are universal conventions in the Python data science community. Using them makes your code immediately recognizable to any other Python practitioner.

If you are working inside a Jupyter Notebook, you should add the magic command %matplotlib inline at the top of your notebook (in its own cell, before any plotting code). This instructs Jupyter to render Matplotlib figures as static images directly within the notebook output cells, rather than opening a separate pop-up window. In newer versions of JupyterLab, this is often the default behavior, but including it explicitly ensures consistent behavior across environments:

%matplotlib inline
import matplotlib.pyplot as plt
import seaborn as sns
import pandas as pd
import numpy as np

An alternative magic command, %matplotlib notebook, enables interactive figures within classic Jupyter Notebook (allowing zooming and panning), though it is less commonly used today since JupyterLab and tools like Plotly handle interactivity differently.

Configuring Plot Styles and Themes

One of the first decisions when building visualizations is choosing an overall visual style. Both Matplotlib and Seaborn offer style configuration systems, and they can be combined.

Seaborn provides five built-in themes that control the background and grid appearance of plots:

Theme Name Description Best Use Case
darkgrid Dark grey background with white grid lines (Seaborn default) General exploratory analysis
whitegrid White background with light grey grid lines Charts where precise value reading matters
dark Dark grey background, no grid lines Presentations, dark-mode displays
white White background, no grid lines Minimal, clean publication figures
ticks White background with tick marks only (no grid) Academic publications, scientific journals

Apply a Seaborn theme using sns.set_theme():

sns.set_theme(style="whitegrid", palette="muted", font_scale=1.2)

The font_scale parameter scales all text elements uniformly, which is useful when preparing figures for posters or slides where larger text is needed. The palette parameter sets the default color cycle for all subsequent plots.

Seaborn also offers a rich color palette system. Palettes can be set globally with sns.set_palette() or passed directly to individual plot functions. Common palettes include:

  • Qualitative palettes (for categorical data): "Set1", "Set2", "tab10", "Paired"
  • Sequential palettes (for ordered data): "Blues", "viridis", "plasma", "YlOrRd"
  • Diverging palettes (for data with a meaningful midpoint): "coolwarm", "RdBu", "vlag"
sns.set_palette("Set2")
# All subsequent plots will use the Set2 color palette

On the Matplotlib side, style sheets offer an alternative or complementary approach to styling. These are accessed via plt.style.use():

plt.style.use("ggplot")      # Mimics the ggplot2 look from R
plt.style.use("fivethirtyeight")  # Mimics FiveThirtyEight journalism style
plt.style.use("seaborn-v0_8")    # Applies a Seaborn-like look via Matplotlib

For fine-grained control over every rendering detail, Matplotlib exposes its global settings through plt.rcParams — a dictionary of configuration parameters that control default figure size, DPI, font sizes, line widths, and hundreds of other properties:

plt.rcParams['figure.figsize'] = (10, 6)    # Default figure size in inches
plt.rcParams['figure.dpi'] = 150            # Resolution (dots per inch)
plt.rcParams['font.size'] = 12             # Base font size
plt.rcParams['axes.titlesize'] = 16        # Title font size
plt.rcParams['lines.linewidth'] = 2.0      # Default line thickness

Setting rcParams at the top of a script establishes consistent formatting for all figures generated in that session, which is a best practice for any project where visual consistency matters.

Understanding the Figure and Axes Structure

No concept is more foundational to working with Matplotlib — and by extension Seaborn — than the Figure and Axes hierarchy. Grasping this structure resolves the majority of confusion that new users experience when trying to customize plots.

A Figure is the top-level container for everything you see when a plot is displayed. Think of it as the blank canvas or the page. The Figure holds the overall dimensions, the background, the title at the figure level, the legend if it is figure-level, and most importantly, one or more Axes objects. You can think of the Figure as the frame, and the Axes as the paintings inside the frame.

An Axes object (note: "Axes" refers to the plot area, not the x/y axis lines specifically) is where actual data visualization happens. Each Axes has its own x-axis, y-axis, tick marks, tick labels, gridlines, title, and drawn data elements. A single Figure can contain a grid of multiple Axes, enabling side-by-side or stacked subplot panels.

The recommended way to create a Figure and Axes together is the plt.subplots() function:

fig, ax = plt.subplots()

# Now 'fig' is the Figure object and 'ax' is the single Axes object
ax.plot([1, 2, 3], [4, 5, 6], color='teal', linewidth=2)
ax.set_title("My Chart")
ax.set_xlabel("X")
ax.set_ylabel("Y")
plt.show()

For multiple subplots, pass row and column counts to plt.subplots():

fig, axes = plt.subplots(nrows=1, ncols=2, figsize=(12, 5))

# axes is now a NumPy array of Axes objects
axes[0].plot([1, 2, 3], [1, 4, 9], color='steelblue')
axes[0].set_title("Left Panel")

axes[1].bar(['A', 'B', 'C'], [10, 25, 15], color='coral')
axes[1].set_title("Right Panel")

plt.tight_layout()
plt.show()

The plt.tight_layout() call is a best practice that automatically adjusts spacing between subplots to prevent labels and titles from overlapping.

Seaborn integrates cleanly with this structure through its ax parameter. Most Seaborn "Axes-level" functions (such as sns.histplot, sns.scatterplot, sns.boxplot) accept an optional ax argument that tells Seaborn which specific Axes to draw into. This is how you place a Seaborn chart into one panel of a multi-panel Matplotlib figure:

import seaborn as sns
import matplotlib.pyplot as plt

df = sns.load_dataset("tips")

fig, axes = plt.subplots(1, 2, figsize=(14, 5))

# Seaborn plot goes into the left Axes
sns.histplot(data=df, x="total_bill", kde=True, ax=axes[0])
axes[0].set_title("Distribution of Total Bill")

# Seaborn plot goes into the right Axes
sns.scatterplot(data=df, x="total_bill", y="tip", hue="sex", ax=axes[1])
axes[1].set_title("Total Bill vs Tip")

plt.tight_layout()
plt.show()

This example demonstrates the full power of the combined approach: Seaborn handles the statistical plotting logic concisely, while Matplotlib's Figure/Axes structure organizes the layout precisely. Understanding this workflow — creating a Figure with plt.subplots(), passing Axes to Seaborn functions via the ax parameter, and then refining with Matplotlib methods — is the single most important pattern for advanced Python visualization.

It is also worth distinguishing between Seaborn's two categories of plotting functions. Axes-level functions (like sns.histplot, sns.scatterplot) draw onto a single Axes and support the ax parameter. Figure-level functions (like sns.displot, sns.relplot, sns.catplot) create their own Figure and manage their own Axes grid internally. Figure-level functions are more powerful for creating faceted, multi-panel plots automatically, but they are less directly compatible with the plt.subplots() workflow. Knowing which type of function you are calling prevents a common source of confusion when combining the two libraries.
