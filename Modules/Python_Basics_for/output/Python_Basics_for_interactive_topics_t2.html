Python Data Types and Structures

▶

Python provides a rich set of built-in data types and structures that form the foundation of every analytical program. Choosing the right type or structure is not merely a stylistic preference — it directly affects correctness, performance, and the ease with which data can be cleaned, transformed, and analyzed. This topic examines the most essential Python data types used in analytics: numeric types (integers and floats), strings, lists, tuples, and dictionaries. For each, we explore how it works internally, how to use it effectively, and when to prefer it over the alternatives.

Numeric Data Types: Integers and Floats

Numbers in Python are represented by two primary types: integers (int) and floating-point numbers (float). Understanding the difference between them is critical for writing accurate analytical code.

Integers store whole numbers — positive, negative, or zero — with no fractional component. Python integers have arbitrary precision, meaning they can grow as large as available memory allows, unlike many lower-level languages that cap integer size at 32 or 64 bits. Integers are the natural choice for counts, array indices, loop counters, and any categorical code that should never be fractional.

# Integer examples
record_count = 1500
year = 2024
customer_id = 40821

print(type(record_count))   # <class 'int'>
print(record_count + 1)     # 1501
print(record_count // 100)  # 15  (integer division)
print(record_count % 7)     #  6  (remainder / modulo)

Floats store numbers with a decimal point and are implemented as IEEE 754 double-precision values, giving roughly 15–17 significant decimal digits of precision. Floats are essential whenever you work with measurements, probabilities, percentages, monetary amounts, or any result of ordinary division.

# Float examples
temperature = 98.6
conversion_rate = 0.034
pi_approx = 3.14159

print(type(temperature))    # <class 'float'>
print(10 / 3)               # 3.3333333333333335  (always float in Python 3)
print(round(10 / 3, 4))    # 3.3333

A key behavior to internalize is type promotion: whenever Python performs arithmetic between an integer and a float, the result is always a float. This happens automatically and silently, which is usually convenient but can mask unexpected type changes in a pipeline.

result = 5 + 2.0   # int + float → float
print(result)      # 7.0
print(type(result))# <class 'float'>

Analysts must also be aware of floating-point precision limitations. Because floats are stored in binary, some decimal fractions cannot be represented exactly, leading to small rounding errors that can accumulate across many calculations.

print(0.1 + 0.2)          # 0.30000000000000004  — not exactly 0.3
print(0.1 + 0.2 == 0.3)   # False

# Use round() or math.isclose() for comparisons
import math
print(math.isclose(0.1 + 0.2, 0.3))  # True

For financial or high-precision work, Python's decimal.Decimal class provides exact decimal arithmetic, though at the cost of some performance. Knowing when to use int, float, or Decimal prevents subtle bugs that are notoriously hard to detect in large datasets.

Strings and Text Data

A string (str) is an ordered sequence of Unicode characters. In Python 3, all strings are Unicode by default, meaning they can represent virtually any character from any human language — an important feature when handling international datasets. Strings are delimited by single quotes ('…'), double quotes ("…"), or triple quotes ("""…""" or '''…''') for multi-line content.

name = 'Alice'
city = "New York"
description = """This dataset contains
customer records from Q1 2024."""

print(type(name))   # <class 'str'>
print(len(city))    # 8
print(city[0])      # N   (zero-based indexing)
print(city[-1])     # k   (negative indexing counts from the end)

Strings are immutable: once created, the characters inside cannot be changed in place. Any operation that appears to "modify" a string actually creates a new string object. This is an important distinction when writing performance-sensitive loops.

greeting = "hello"
# greeting[0] = "H"   # ← raises TypeError — strings are immutable

greeting = greeting.capitalize()  # creates a NEW string
print(greeting)   # Hello

Python includes a large library of built-in string methods that are invaluable for data cleaning:

  • .strip() / .lstrip() / .rstrip() — remove leading and/or trailing whitespace, fixing a common data-entry problem.
  • .lower() / .upper() / .title() — normalize case before comparisons or grouping operations.
  • .split(delimiter) — breaks a string into a list of substrings, useful for parsing CSV-style fields or tokenizing text.
  • .replace(old, new) — substitutes substrings, helpful for removing unwanted characters.
  • .startswith() / .endswith() — boolean checks for filtering rows based on string patterns.
  • .find() / .count() — locate or count occurrences of a substring within a larger string.
raw = "  Sales_Region_North  "
cleaned = raw.strip().lower().replace("_", " ")
print(cleaned)   # "sales region north"

csv_row = "Alice,28,New York,Engineer"
fields = csv_row.split(",")
print(fields)    # ['Alice', '28', 'New York', 'Engineer']
print(fields[3]) # Engineer

String formatting is essential for constructing readable output from variable data. The modern, preferred approach is f-strings (formatted string literals), introduced in Python 3.6. An f-string is prefixed with f, and expressions inside curly braces {} are evaluated and embedded directly into the string.

name = "Bob"
score = 94.756

# f-string with format specifier for 2 decimal places
message = f"Analyst {name} achieved a score of {score:.2f} out of 100."
print(message)  # Analyst Bob achieved a score of 94.76 out of 100.

# Embedding calculations directly
items = 320
rate = 0.08
print(f"Tax on {items} items: ${items * rate:.2f}")  # Tax on 320 items: $25.60

Older alternatives include str.format() and %-formatting, which you may encounter in legacy code, but f-strings are cleaner and faster for new work.

Lists: Ordered and Mutable Collections

A list is Python's most versatile built-in data structure. It is an ordered, mutable sequence that can hold items of any type — including mixed types — in a single container. Lists are defined with square brackets, with items separated by commas.

scores = [88, 92, 75, 100, 67]
mixed  = [1, "Alice", 3.14, True, None]
nested = [[1, 2], [3, 4], [5, 6]]   # list of lists (like a 2-D table)

print(type(scores))   # <class 'list'>
print(len(scores))    # 5

Indexing and slicing are the primary ways to access list elements. Python uses zero-based indexing, so the first element is at index 0. Negative indices count backwards from the end. Slicing uses the syntax list[start:stop:step] and returns a new list containing the selected elements (the stop index is exclusive).

scores = [88, 92, 75, 100, 67]

print(scores[0])      # 88    — first element
print(scores[-1])     # 67    — last element
print(scores[1:4])    # [92, 75, 100]  — indices 1, 2, 3
print(scores[::2])    # [88, 75, 67]   — every other element
print(scores[::-1])   # [67, 100, 75, 92, 88]  — reversed

Because lists are mutable, you can modify them in place using direct assignment or built-in methods:

  • .append(item) — adds a single item to the end of the list.
  • .extend(iterable) — appends all items from another iterable.
  • .insert(index, item) — inserts an item at a specified position.
  • .remove(value) — removes the first occurrence of a value.
  • .pop(index) — removes and returns the item at the given index (default: last).
  • .sort() — sorts the list in place; use sorted() to return a new sorted list.
  • .reverse() — reverses the list in place.
  • len(list) — returns the number of items.
regions = ["North", "South", "East"]
regions.append("West")
print(regions)          # ['North', 'South', 'East', 'West']

regions.remove("South")
print(regions)          # ['North', 'East', 'West']

regions.sort()
print(regions)          # ['East', 'North', 'West']

popped = regions.pop(0)
print(popped)           # East
print(regions)          # ['North', 'West']

List comprehensions are a concise, Pythonic way to build or transform lists in a single readable expression, avoiding verbose for-loops:

raw_scores = [88, 92, 75, 100, 67]

# Normalize each score to a 0–1 scale
normalized = [s / 100 for s in raw_scores]
print(normalized)   # [0.88, 0.92, 0.75, 1.0, 0.67]

# Filter only passing scores (>= 70)
passing = [s for s in raw_scores if s >= 70]
print(passing)      # [88, 92, 75, 100]

Tuples: Ordered and Immutable Sequences

A tuple looks and behaves much like a list, but with one fundamental difference: it is immutable. Once a tuple is created, its contents cannot be changed — no adding, removing, or reassigning elements. Tuples are defined using parentheses (though the parentheses are technically optional; it is the comma that makes a tuple).

coordinates = (40.7128, -74.0060)   # latitude, longitude of New York
rgb_red     = (255, 0, 0)
single      = (42,)                 # trailing comma required for single-element tuple

print(type(coordinates))   # <class 'tuple'>
print(coordinates[0])      # 40.7128
print(len(rgb_red))        # 3

# Attempting to modify raises an error:
# coordinates[0] = 51.5   # TypeError: 'tuple' object does not support item assignment

Tuples support the same indexing and slicing syntax as lists, making element access straightforward:

point = (10, 20, 30, 40, 50)
print(point[2])     # 30
print(point[1:4])   # (20, 30, 40)
print(point[::-1])  # (50, 40, 30, 20, 10)

Tuple unpacking is a powerful feature that assigns each element of a tuple to a separate variable in a single statement. It is frequently used to return multiple values from a function or to destructure rows of data:

employee = ("Diana", "Engineering", 95000)
name, department, salary = employee   # unpacking

print(name)        # Diana
print(department)  # Engineering
print(salary)      # 95000

# Swap two variables elegantly using tuple packing/unpacking
a, b = 10, 20
a, b = b, a
print(a, b)        # 20 10

Because tuples are immutable, Python can optimize their storage and access slightly better than lists. They also serve as valid dictionary keys (since they are hashable), whereas lists cannot. This makes tuples the right choice for composite keys in lookup tables — for example, (country, year) as a key to retrieve a GDP value.

gdp = {
    ("USA", 2022): 25.46,
    ("DEU", 2022): 4.07,
    ("JPN", 2022): 4.23
}
print(gdp[("USA", 2022)])   # 25.46

Use tuples to represent records with fixed fields (e.g., a database row, a geographic coordinate, an RGB color) or any data that should be treated as a single, unchangeable unit. The immutability acts as self-documenting intent: any reader of the code understands that this data should not change.

Dictionaries: Key-Value Data Storage

A dictionary (dict) stores data as key-value pairs. It is the go-to structure when your data has named attributes — think of it as a flexible, in-memory record or row from a database table. Dictionaries are defined with curly braces; each pair is written as key: value, separated by commas. Keys must be unique within a dictionary and must be of an immutable type (strings, numbers, and tuples all work; lists do not).

customer = {
    "id": 40821,
    "name": "Alice Chen",
    "region": "West",
    "annual_spend": 12450.75,
    "active": True
}

print(type(customer))           # <class 'dict'>
print(customer["name"])         # Alice Chen
print(customer["annual_spend"]) # 12450.75

Values are retrieved by specifying the key in square brackets. Using a key that does not exist raises a KeyError; to avoid this, use the safer .get(key, default) method, which returns None (or a specified default) if the key is absent.

print(customer.get("email"))              # None  (key doesn't exist, no error)
print(customer.get("email", "N/A"))       # N/A

Dictionaries are mutable: keys and values can be added, updated, or removed at any time:

customer["email"] = "alice@example.com"   # add a new key
customer["annual_spend"] = 13200.00       # update an existing value
del customer["active"]                    # remove a key-value pair

print(customer)
# {'id': 40821, 'name': 'Alice Chen', 'region': 'West',
#  'annual_spend': 13200.0, 'email': 'alice@example.com'}

Python dictionaries (since version 3.7) maintain insertion order, so iterating over a dictionary visits keys in the order they were added. Three essential iteration methods are:

  • .keys() — returns a view of all keys.
  • .values() — returns a view of all values.
  • .items() — returns a view of (key, value) tuples, most commonly used in loops.
for key, value in customer.items():
    print(f"{key}: {value}")

# id: 40821
# name: Alice Chen
# region: West
# annual_spend: 13200.0
# email: alice@example.com

Dictionaries offer O(1) average-time complexity for lookups, insertions, and deletions, achieved through an internal hash table. This makes them dramatically faster than scanning a list when you need to find a value by name. A common analytics pattern is building a frequency dictionary (word count, category count) by incrementing values as data is processed:

transactions = ["North", "South", "North", "West", "North", "South"]

region_counts = {}
for region in transactions:
    region_counts[region] = region_counts.get(region, 0) + 1

print(region_counts)   # {'North': 3, 'South': 2, 'West': 1}

Dictionaries can also be nested — values can themselves be dictionaries — enabling hierarchical data representations similar to JSON:

company = {
    "name": "DataCorp",
    "headquarters": {"city": "Chicago", "state": "IL"},
    "employee_count": 450
}
print(company["headquarters"]["city"])   # Chicago

Choosing the Right Data Structure for Analytics

Selecting the most appropriate data structure is a skill that separates readable, efficient analytics code from code that is fragile or slow. The decision often comes down to three properties: mutability (can the data change?), ordering (does sequence matter?), and access pattern (do you look up by position or by name?).

Structure Ordered? Mutable? Duplicates? Lookup By Typical Use Case
list Yes Yes Yes Integer index Sequences to iterate, filter, or sort
tuple Yes No Yes Integer index Fixed records, function return values, dict keys
dict Yes (insertion order) Yes Keys: No; Values: Yes Named key Records with named fields, fast lookups, frequency counts
str Yes No Yes Integer index Text data, labels, messages

To make these trade-offs concrete, consider a practical scenario: you are processing a dataset of customer transactions.

  • Use a list to hold all transaction amounts when you need to iterate through them, compute a sum, filter by threshold, or sort by value. The mutability lets you append new transactions as they arrive.
  • Use a tuple to represent each individual transaction as an immutable record — for example, (transaction_id, date, amount, region) — where the structure is fixed and you want to prevent accidental modification. Tuples can also serve as composite dictionary keys, like (customer_id, product_id).
  • Use a dictionary to map customer IDs to their cumulative spend, or to store a single customer's attributes (name, region, tier). The O(1) lookup means you can check or update any customer's record instantly, regardless of how many customers are in the dataset.
# Practical example combining all three structures

# Raw data as a list of tuples (immutable records)
transactions = [
    (1001, "Alice", "North", 250.00),
    (1002, "Bob",   "South", 480.50),
    (1003, "Alice", "North", 130.75),
    (1004, "Carol", "West",  920.00),
    (1005, "Bob",   "South",  60.25),
]

# Aggregate spend per customer using a dictionary
spend_by_customer = {}
for txn_id, customer, region, amount in transactions:   # tuple unpacking
    spend_by_customer[customer] = spend_by_customer.get(customer, 0) + amount

print(spend_by_customer)
# {'Alice': 380.75, 'Bob': 540.75, 'Carol': 920.0}

# Find the top spender
top_customer = max(spend_by_customer, key=spend_by_customer.get)
print(f"Top spender: {top_customer} (${spend_by_customer[top_customer]:.2f})")
# Top spender: Carol ($920.00)

# Collect amounts as a list for statistical operations
amounts = [t[3] for t in transactions]   # list comprehension
print(f"Average transaction: ${sum(amounts)/len(amounts):.2f}")
# Average transaction: $368.30

This example illustrates how the three structures complement each other in real analytical workflows: tuples provide safe, structured raw records; lists support iteration and aggregation; and dictionaries enable fast, named lookups and accumulation. Developing fluency with all three — and understanding when each earns its place — is a foundational skill for any Python data analyst.
