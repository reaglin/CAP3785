Module Topics

Introduction to Pivot Tables

Students learn the fundamentals of pivot tables, including how to create, configure, and navigate them to summarize and explore large datasets efficiently.
  • What Is a Pivot Table? — A pivot table is an interactive Excel tool that allows users to summarize, group, and analyze large datasets without writing formulas or altering the original data.
  • Preparing Your Data for a Pivot Table — Before creating a pivot table, the source data must be properly structured to ensure accurate and reliable results.
  • Creating a Pivot Table — Inserting a pivot table in Excel is a straightforward process that begins by selecting your data source and choosing where the pivot table will be placed.
  • Understanding the PivotTable Fields Pane — The PivotTable Fields pane is the control center for configuring a pivot table, allowing users to assign data fields to specific areas of the report.
  • Configuring Rows, Columns, and Values — Assigning fields to the Rows, Columns, and Values areas defines the structure and content of the pivot table summary.
  • Navigating and Refreshing a Pivot Table — Once created, pivot tables offer interactive navigation features that allow users to drill down into data and keep summaries up to date.
  • Changing Summary Functions and Formatting Values — By default, Excel uses SUM for numeric fields and COUNT for text fields, but these aggregation functions can be changed to better suit the analysis goal.

Advanced Pivot Table Techniques

This topic covers advanced pivot table features such as calculated fields, grouping, filtering, and slicers to enable deeper and more flexible data analysis.
  • Calculated Fields in Pivot Tables — Calculated fields allow you to create new data columns within a pivot table by writing custom formulas based on existing fields, without altering the source data.
  • Grouping Data in Pivot Tables — Grouping enables you to consolidate rows or columns into meaningful categories, making large datasets easier to analyze at different levels of granularity.
  • Filtering Data Within Pivot Tables — Pivot table filters let you narrow the displayed data to specific subsets, helping you focus your analysis on relevant records.
  • Using Slicers for Interactive Filtering — Slicers are visual, clickable filter controls that make it easy to interactively segment pivot table data without navigating dropdown menus.
  • Timeline Slicers for Date-Based Filtering — Timeline slicers are a specialized slicer type designed specifically for filtering date fields, providing an intuitive scrollable time bar.
  • Drilling Down and Expanding Pivot Table Data — Pivot tables support multi-level hierarchies that allow you to expand or collapse data levels for top-down exploration of your dataset.
  • Refreshing and Managing Pivot Table Data Sources — Keeping pivot tables up to date requires understanding how to refresh data and manage the connection to the underlying data source.

Building Dynamic Dashboards

Students learn how to design and assemble interactive Excel dashboards by combining pivot tables, charts, and visual elements into a cohesive reporting layout.
  • Planning the Dashboard Layout — Before building a dashboard, effective planning ensures all required information is organized logically and the layout serves the intended audience.
  • Connecting Pivot Tables to Dashboard Charts — Pivot charts are directly linked to pivot tables, making them the ideal chart type for a dynamic dashboard because they update automatically when data or filters change.
  • Adding Slicers for Interactivity — Slicers are visual filter controls that allow dashboard users to segment data with a single click, making the dashboard interactive without requiring any formula knowledge.
  • Using Timeline Slicers for Date Filtering — Timeline slicers provide a specialized interactive control for filtering data by date periods such as months, quarters, or years.
  • Incorporating Summary KPI Cells — Key performance indicator cells display single high-level metrics such as total revenue, average order value, or year-over-year growth prominently on the dashboard.
  • Formatting and Polishing the Dashboard — A well-formatted dashboard communicates professionalism and makes data easier to interpret by applying consistent design principles throughout the layout.
  • Enabling One-Click Data Refresh — A truly dynamic dashboard updates all its pivot tables and charts instantly when the underlying data changes, requiring minimal manual intervention from the user.

Dashboard Interactivity and Visual Design

This topic focuses on enhancing dashboards with slicers, timelines, and formatting best practices to make them intuitive and visually compelling for end users.
  • Adding Slicers to Dashboards — Slicers are visual filter buttons that allow users to interactively filter pivot table data with a single click, making dashboards more intuitive and user-friendly.
  • Using Timelines for Date-Based Filtering — Timelines are a specialized type of slicer designed specifically for date fields, enabling users to filter data by time periods such as months, quarters, or years.
  • Laying Out a Dashboard for Usability — A well-structured dashboard layout guides the user's eye naturally through the most important information, placing key metrics prominently and supporting details below or to the side.
  • Applying Consistent Color Schemes and Fonts — Visual consistency in color and typography signals professionalism and helps users focus on data rather than being distracted by design inconsistencies.
  • Formatting Charts for Clarity — Chart formatting choices such as removing gridlines, simplifying legends, and adding data labels directly to chart elements can significantly improve how quickly users interpret data.
  • Protecting and Finalizing the Dashboard — Once a dashboard is complete, protecting certain elements prevents accidental edits while still allowing users to interact with slicers, timelines, and filters.

Introduction to Power Query

Students are introduced to Power Query as a tool for importing data from multiple sources and automating the process of connecting Excel to external datasets.
  • What Is Power Query? — Power Query is a built-in Excel tool designed to help users import, connect, and transform data from a wide variety of sources with minimal manual effort.
  • Supported Data Sources — Power Query can connect Excel to a broad range of external data sources, making it a versatile tool for data import workflows.
  • Connecting Excel to an External Dataset — Establishing a connection to an external dataset is the first step in using Power Query, and it is done through a guided interface that requires no coding.
  • The Power Query Editor — Once a data source is connected, the Power Query Editor opens as a dedicated workspace where users can inspect and begin transforming their data.
  • Automating Data Import with Refresh — One of Power Query's most powerful features is the ability to refresh data connections, automatically re-importing and re-transforming updated data from the source.
  • Why Power Query Improves Data Workflows — Power Query transforms the data import process from a repetitive manual task into a repeatable, auditable workflow that scales with the complexity of the data.

Data Transformation with Power Query

This topic covers how to clean, reshape, and transform raw data using Power Query's built-in tools, reducing the need for manual data preparation steps.
  • Introduction to the Power Query Editor — The Power Query Editor is Excel's dedicated environment for loading, cleaning, and shaping data before it enters your worksheets or data model.
  • Connecting to and Importing Raw Data Sources — Power Query can connect to a wide variety of data sources, including Excel files, CSV files, databases, web pages, and cloud services.
  • Cleaning Data: Removing Errors, Duplicates, and Blanks — One of Power Query's most practical uses is automating the removal of common data quality issues such as duplicate rows, blank cells, and error values.
  • Reshaping Data: Splitting, Merging, and Pivoting Columns — Power Query provides tools to restructure columns so that raw data conforms to the layout needed for analysis, without altering the source file.
  • Changing Data Types and Formatting Values — Assigning correct data types to each column ensures that Excel treats numbers, dates, and text appropriately during analysis and visualization.
  • Adding Custom and Conditional Columns — Power Query allows you to create new calculated columns based on existing data, using either a formula or a set of conditional rules — without writing complex Excel formulas on the sheet.
  • Refreshing Queries and Automating Data Preparation — Because Power Query records every transformation as a repeatable sequence of steps, the entire data preparation workflow reruns automatically whenever the source data changes.

Automating Refresh Workflows

Students learn how to configure Power Query to refresh data automatically, ensuring that pivot tables and dashboards always reflect the most current information.
  • Understanding Data Refresh in Power Query — Data refresh is the process of updating a Power Query connection so that the latest source data is pulled into Excel, keeping pivot tables and dashboards current.
  • Configuring Connection Properties for Automatic Refresh — Excel's Connection Properties dialog allows you to set a query to refresh at a defined time interval or whenever the workbook is opened.
  • Refresh All: Updating Multiple Queries at Once — The Refresh All command on the Data tab triggers a simultaneous refresh of every Power Query connection and pivot table in the workbook.
  • Linking Pivot Tables to Power Query Outputs — For automatic refresh to flow through to pivot tables and dashboards, pivot tables must be connected to the same data model or table that Power Query loads into.
  • Automating Refresh with VBA Macros — VBA macros can be used to trigger Power Query refreshes programmatically, enabling event-driven or scheduled automation within Excel.
  • Managing Background Refresh and Performance — Background refresh allows Excel to continue responding to user input while a query updates, but it requires careful management to avoid conflicts in automated workflows.
  • Testing and Validating Refresh Workflows — After configuring automatic refresh settings, it is essential to test the full workflow to confirm that updated source data correctly propagates to dashboards and pivot tables.
