File Input and Output for Data

▶

One of the most essential skills in data analytics with Python is the ability to read from and write to external files. Real-world data rarely lives inside a Python script — it arrives as text files, spreadsheets exported as CSV, log files, or structured plain-text reports. Understanding how Python interacts with the file system allows you to load that data into memory, process and clean it, and save your results back to disk. This topic walks through every key aspect of file I/O: opening and safely closing files, reading content in different ways, writing output, working with the widely used CSV format, cleaning raw data as it is read, and navigating the file system reliably across operating systems.

Opening and Closing Text Files

Before you can read from or write to a file, Python must open a connection to it. The built-in open() function establishes that connection and returns a file object through which all subsequent operations are performed. The two most important arguments to open() are the file path and the mode, which tells Python what you intend to do with the file.

Mode Symbol Description
Read 'r' Opens an existing file for reading. Raises FileNotFoundError if the file does not exist.
Write 'w' Opens a file for writing. Creates the file if it does not exist; overwrites all existing content if it does.
Append 'a' Opens a file for writing, but preserves existing content and adds new data at the end.
Read + Write 'r+' Opens an existing file for both reading and writing without truncating it.

The simplest way to open a file is a direct call to open(), assigning the result to a variable:

f = open('sales.txt', 'r')
content = f.read()
f.close()

Notice the explicit call to f.close(). Closing a file releases the operating system resource tied to it. If you skip this step, several problems can follow: data you intended to write may not be flushed from the buffer to disk, the file may appear locked to other programs or processes, and in long-running scripts you may exhaust the limit of simultaneously open file handles. These bugs are particularly insidious because they often go unnoticed in small scripts yet cause failures in production environments.

The recommended and far safer approach is the with statement, also called a context manager:

with open('sales.txt', 'r') as f:
    content = f.read()
# The file is automatically closed here, even if an exception occurred inside the block

When execution leaves the with block — whether normally or because an exception was raised — Python automatically calls f.close() on your behalf. This removes an entire category of resource-leak bugs and is the standard idiom you will see in professional Python code.

A common beginner mistake is providing the wrong file path. If 'sales.txt' does not exist in the directory from which the script is run, Python raises a FileNotFoundError. Always double-check that your path is correct relative to your working directory, or use an absolute path. The section on file paths later in this topic explains both strategies in depth.

Reading Data from Text Files

Once a file is open in read mode, Python provides several methods for extracting its contents. The right choice depends on the size of the file and what you plan to do with the data.

Reading the entire file at once with .read()

Calling f.read() loads the complete file into a single string:

with open('notes.txt', 'r') as f:
    text = f.read()

print(text)

This is convenient for small configuration files or documents you want to search with string methods, but it is dangerous with large files because the entire content is held in memory simultaneously. A 2 GB log file would consume 2 GB of RAM the instant .read() is called.

Reading one line at a time with .readline()

f.readline() reads and returns the next line from the file, including its trailing newline character '\n'. Calling it repeatedly advances through the file line by line:

with open('notes.txt', 'r') as f:
    first_line  = f.readline()
    second_line = f.readline()

print(first_line.strip())
print(second_line.strip())

When the end of the file is reached, .readline() returns an empty string '', which can be used to control a loop. This method is useful when you need fine-grained, one-at-a-time control over reading.

Reading all lines into a list with .readlines()

f.readlines() reads every line and returns them as a Python list, where each element is a string representing one line, newline character included:

with open('notes.txt', 'r') as f:
    lines = f.readlines()

print(lines)
# ['First line\n', 'Second line\n', 'Third line\n']

This is handy when you need random access to lines by index, but like .read() it loads everything into memory at once.

Iterating directly over the file object

The most memory-efficient strategy for large files is to iterate over the file object itself in a for loop. Python reads one line at a time from disk, keeping only that single line in memory:

with open('large_dataset.txt', 'r') as f:
    for line in f:
        processed = line.strip()
        print(processed)

This pattern scales to files of any size and is the preferred approach in data analytics when you cannot guarantee the file will fit comfortably in RAM.

Stripping newlines and whitespace

Every line read from a text file retains its trailing newline character '\n'. If you compare or convert that line without removing it, you can introduce subtle bugs. The .strip() string method removes all leading and trailing whitespace — including '\n', '\r', tabs, and spaces — making it essential for cleaning raw lines:

raw_line = '  42.5\n'
clean    = raw_line.strip()   # '42.5'
value    = float(clean)       # 42.5

Use .lstrip() or .rstrip() if you only want to strip from one side.

Writing Data to Text Files

Writing to a file follows the same open() pattern, but with mode 'w' or 'a'. The two primary methods are .write() and .writelines().

Writing a string with .write()

with open('output.txt', 'w') as f:
    f.write('Hello, world!')

Unlike the built-in print() function, .write() does not append a newline automatically. If you write multiple lines without explicitly including '\n', the text will run together on a single line in the file:

# Incorrect — all text ends up on one line
with open('output.txt', 'w') as f:
    f.write('Line one')
    f.write('Line two')

# Correct — each line is separated properly
with open('output.txt', 'w') as f:
    f.write('Line one\n')
    f.write('Line two\n')

Opening in 'w' mode creates the file if it does not already exist, which is extremely convenient for generating new output or report files programmatically. Be aware, however, that if the file does already exist, all of its contents are erased immediately when the file is opened — not when you call .write(). Use 'a' (append) mode when you want to preserve existing content and add to it:

with open('log.txt', 'a') as f:
    f.write('Session ended at 14:32\n')

Writing a list of strings with .writelines()

.writelines() accepts any iterable of strings and writes them all to the file in one call. It is a convenient shortcut when you have already assembled your lines into a list:

lines = ['apple\n', 'banana\n', 'cherry\n']

with open('fruits.txt', 'w') as f:
    f.writelines(lines)

Like .write(), .writelines() does not add newlines for you — each string in the list must already contain '\n' if you want line breaks in the file. A common pattern is to build the list with newlines included, or to use a list comprehension:

items = ['apple', 'banana', 'cherry']

with open('fruits.txt', 'w') as f:
    f.writelines(item + '\n' for item in items)

Working with CSV Files Using Python's csv Module

Comma-Separated Values (CSV) is one of the most universally used formats for tabular data. Every major spreadsheet application and database tool can export to CSV, making it the lingua franca of data exchange. While you could technically parse a CSV by splitting each line on commas, this naive approach breaks immediately when field values contain commas, quotation marks, or embedded newlines. Python's built-in csv module handles all of these edge cases correctly.

Reading a CSV with csv.reader()

import csv

with open('sales.csv', 'r', newline='') as f:
    reader = csv.reader(f)
    for row in reader:
        print(row)

Each row yielded by the reader is a plain Python list of strings, where each element corresponds to one column in that row. For example, a CSV file containing:

product,quantity,price
apple,100,0.50
banana,80,0.30

Would produce the following rows:

['product', 'quantity', 'price']
['apple', '100', '0.50']
['banana', '80', '0.30']

Notice that all values are strings — '100' not 100, and '0.50' not 0.50. Type conversion is always your responsibility and is covered in the parsing section below.

The newline='' argument inside open() is important, particularly on Windows. Without it, the CSV module can misinterpret the line endings used inside quoted fields, or insert extra blank rows into output. The csv module documentation explicitly recommends always passing newline=''.

Skipping the header row

When the first row contains column names rather than data, use next() to advance the reader past it before your main loop:

import csv

with open('sales.csv', 'r', newline='') as f:
    reader = csv.reader(f)
    headers = next(reader)          # consumes and returns the header row
    for row in reader:
        product  = row[0]
        quantity = int(row[1])
        price    = float(row[2])
        print(f'{product}: {quantity} units at ${price:.2f}')

Reading with csv.DictReader()

csv.DictReader() is a powerful alternative that reads each row as a Python dictionary, where the keys are taken from the header row automatically:

import csv

with open('sales.csv', 'r', newline='') as f:
    reader = csv.DictReader(f)
    for row in reader:
        print(row['product'], row['quantity'], row['price'])

This makes column access by name rather than by position, which produces much more readable and maintainable code. If a colleague adds a column to the CSV, positional indexing silently breaks; named access with DictReader continues to work correctly as long as the column name is unchanged.

Writing a CSV with csv.writer()

import csv

rows = [
    ['product', 'quantity', 'price'],
    ['apple',   100,        0.50],
    ['banana',  80,         0.30],
]

with open('output.csv', 'w', newline='') as f:
    writer = csv.writer(f)
    for row in rows:
        writer.writerow(row)

writer.writerow() accepts a list and writes it as one CSV row, handling quoting and escaping automatically. You can also write all rows at once with writer.writerows(rows).

If you are working with a list of dictionaries — a common result of API calls or data transformation — csv.DictWriter is the matching counterpart to DictReader:

import csv

data = [
    {'product': 'apple',  'quantity': 100, 'price': 0.50},
    {'product': 'banana', 'quantity': 80,  'price': 0.30},
]

with open('output.csv', 'w', newline='') as f:
    fieldnames = ['product', 'quantity', 'price']
    writer = csv.DictWriter(f, fieldnames=fieldnames)
    writer.writeheader()        # writes the header row from fieldnames
    writer.writerows(data)

Parsing and Cleaning Data During File Reading

Raw data read from a file is always text. For analytical work, numeric fields must be converted to int or float, and messy real-world datasets will inevitably contain missing values, inconsistent spacing, or placeholder strings. Building robust parsing logic into your file-reading loop is a foundational data analytics skill.

Type conversion

Apply the appropriate constructor function to each field after reading it:

import csv

with open('sales.csv', 'r', newline='') as f:
    reader = csv.DictReader(f)
    for row in reader:
        product  = row['product'].strip()
        quantity = int(row['quantity'])
        price    = float(row['price'])
        total    = quantity * price
        print(f'{product}: ${total:.2f}')

Handling missing or invalid values

If a field might be empty or contain a placeholder like 'N/A' or 'null', attempting a direct conversion will raise a ValueError and crash your script. Always guard conversions with a check:

raw_quantity = row['quantity'].strip()

if raw_quantity == '' or raw_quantity.lower() == 'n/a':
    quantity = 0          # substitute a safe default
else:
    quantity = int(raw_quantity)

For more compact code, a try/except block achieves the same goal:

try:
    quantity = int(row['quantity'].strip())
except ValueError:
    quantity = 0

Skipping malformed rows

Some rows may be corrupted, have too few columns, or contain unexpected content. You can skip them with a continue statement inside a conditional check:

import csv

with open('data.csv', 'r', newline='') as f:
    reader = csv.reader(f)
    next(reader)   # skip header
    for row in reader:
        if len(row) < 3:
            continue   # skip rows that don't have enough columns
        product, quantity_str, price_str = row[0], row[1], row[2]
        # ... rest of processing

Stripping whitespace from every field

It is good practice to apply .strip() to every string field, not just those you suspect have whitespace. Invisible leading or trailing spaces cause string comparisons to fail silently and prevent numeric conversions:

product = row['product'].strip()
# 'apple ' and 'apple' would otherwise be treated as different values

When using csv.reader, you can create a simple helper to strip all fields in a row at once:

cleaned_row = [field.strip() for field in row]

File Paths and Directory Navigation

Every file operation requires Python to locate the file on disk. Understanding paths is critical for writing scripts that work reliably across different machines and operating systems.

Relative paths

A relative path is interpreted starting from the current working directory — the folder from which the Python script is launched, not necessarily the folder where the script file lives. For example:

with open('data/sales.csv', 'r') as f:
    ...

This tells Python to look for a data folder inside the current working directory, then open sales.csv inside it. If you run your script from a different directory, the path breaks. Relative paths are compact and portable, but you must be aware of where your script is being run from.

Absolute paths

An absolute path specifies the full location from the root of the file system. It works regardless of the current working directory:

# Windows
with open('C:/Users/alice/project/data/sales.csv', 'r') as f:
    ...

# macOS / Linux
with open('/home/alice/project/data/sales.csv', 'r') as f:
    ...

Absolute paths are reliable on the machine where they were written, but break immediately when the project is moved to a different machine or user account. They are best used for quick debugging, not for code you intend to share.

Building portable paths with os.path.join()

The safest approach for paths that must work on any operating system is to use the os module's os.path.join() function. Windows uses backslashes (\) as path separators while macOS and Linux use forward slashes (/). Hardcoding either in your string will cause failures on the other OS. os.path.join() inserts the correct separator automatically:

import os

path = os.path.join('data', 'sales.csv')
# On Windows: 'data\\sales.csv'
# On macOS/Linux: 'data/sales.csv'

with open(path, 'r') as f:
    ...

For scripts that need to reference files relative to their own location rather than the working directory, combine os.path.join() with __file__ and os.path.dirname():

import os

script_dir = os.path.dirname(os.path.abspath(__file__))
data_path  = os.path.join(script_dir, 'data', 'sales.csv')

with open(data_path, 'r') as f:
    ...

This makes the path relative to the script itself, so it works correctly no matter what directory the user runs the script from.

Checking the current working directory with os.getcwd()

When a FileNotFoundError is raised and you are unsure why, print the current working directory to understand Python's frame of reference:

import os

print(os.getcwd())
# Example output: '/home/alice/my_project'

This is an invaluable debugging step. Once you know the working directory, you can determine whether your relative path points to the correct location or needs adjustment.

Checking whether a file exists before opening it

To avoid a FileNotFoundError when a file's presence is uncertain, use os.path.exists():

import os

path = os.path.join('data', 'sales.csv')

if os.path.exists(path):
    with open(path, 'r') as f:
        content = f.read()
else:
    print(f'File not found: {path}')

These skills are the starting point for every real-world data pipeline you will build.
