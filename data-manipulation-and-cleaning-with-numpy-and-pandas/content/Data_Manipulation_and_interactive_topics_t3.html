Indexing and Selecting Data

▶

Efficiently accessing, selecting, and modifying subsets of data is one of the most fundamental skills in any data analysis workflow. Pandas provides a rich and expressive set of tools for indexing and selecting data from DataFrames and Series. Whether you need a single cell, a range of rows, a handful of columns, or rows that satisfy a complex logical condition, Pandas has a precise mechanism for it. Understanding the distinctions between label-based indexing, position-based indexing, and boolean filtering — and knowing when to use each — is what separates confident, bug-free data manipulation from brittle, unpredictable code.

A DataFrame can be thought of as a two-dimensional labeled table where every row has an index label and every column has a column name. Indexing operations let you reach into that table along both axes simultaneously. The three primary gateways into a DataFrame are loc (label-based), iloc (position-based), and boolean filtering with conditional expressions. Each has its own rules, and mixing them up is one of the most common sources of subtle bugs in pandas code.

Label-Based Indexing with loc

loc is Pandas' label-based indexer. It always works with the actual index labels of the DataFrame — the values you see in the index, not their physical positions in memory. This distinction matters enormously when your index contains strings, dates, or non-sequential integers.

The general syntax is df.loc[row_selector, column_selector]. Each selector can be:

  • A single label — e.g., df.loc[3, 'name'] retrieves the value in row labeled 3 and column 'name'.
  • A list of labels — e.g., df.loc[[1, 3, 5], ['name', 'age']] returns a DataFrame with those specific rows and columns.
  • A label-based slice — e.g., df.loc['a':'d', :] returns all rows whose labels fall between 'a' and 'd' inclusive.
  • A boolean array or Series — passed as the row selector to filter rows based on a condition.

The most important rule about loc slicing: the stop label is inclusive. This is unlike Python's built-in slicing where list[0:3] gives you elements at positions 0, 1, and 2 (not 3). With loc, df.loc[0:3] returns rows labeled 0, 1, 2, and 3.

import pandas as pd

data = {
    'name': ['Alice', 'Bob', 'Carol', 'David', 'Eve'],
    'age':  [25, 30, 35, 40, 45],
    'score': [88, 92, 79, 95, 84]
}
df = pd.DataFrame(data, index=['a', 'b', 'c', 'd', 'e'])

# Single row and column
print(df.loc['b', 'age'])      # 30

# List of rows, single column
print(df.loc[['a', 'c', 'e'], 'score'])
# a    88
# c    79
# e    84

# Label slice — 'b' through 'd' inclusive
print(df.loc['b':'d', 'name':'age'])
#     name  age
# b    Bob   30
# c  Carol   35
# d  David   40

When the column selector is omitted — written as df.loc['b':'d'] — all columns are returned, equivalent to writing df.loc['b':'d', :]. Using loc with a boolean Series is covered in detail under conditional filtering below.

Position-Based Indexing with iloc

iloc is the positional counterpart to loc. It works exclusively with zero-based integer positions, completely ignoring whatever labels the index or columns happen to carry. This makes it behave exactly like indexing a NumPy array or a Python list.

The syntax is df.iloc[row_selector, column_selector] where selectors are integers, lists of integers, or integer slices. Critically, iloc slicing follows standard Python exclusive-stop rules: df.iloc[0:3] gives rows at positions 0, 1, and 2, but not position 3.

# Using the same df from above (index: 'a','b','c','d','e')

# Row at position 0, column at position 1
print(df.iloc[0, 1])    # 25  (Alice's age)

# First three rows, last two columns
print(df.iloc[0:3, 1:3])
#    age  score
# a   25     88
# b   30     92
# c   35     79

# Specific rows and columns by position
print(df.iloc[[0, 2, 4], [0, 2]])
#     name  score
# a  Alice     88
# c  Carol     79
# e    Eve     84

iloc shines in programmatic contexts — looping over rows in chunks, selecting every other row with a step (df.iloc[::2]), or writing general-purpose functions that should not depend on specific label names. It is also the correct tool when you receive a DataFrame with a non-default index (for instance, one set to customer IDs) and you still want to access the literal first row regardless of its label.

Conditional (Boolean) Filtering

Boolean filtering is how you select rows that satisfy logical criteria — the pandas equivalent of a SQL WHERE clause. The mechanism has two steps: first, create a boolean Series by applying a condition to a column; second, pass that Series back into the DataFrame to keep only the rows where the value is True.

# Rows where age is greater than 30
mask = df['age'] > 30
print(mask)
# a    False
# b    False
# c     True
# d     True
# e     True
# Name: age, dtype: bool

filtered = df[mask]
# equivalently: df[df['age'] > 30]
print(filtered)
#     name  age  score
# c  Carol   35     79
# d  David   40     95
# e    Eve   45     84

When you need to combine multiple conditions, each individual condition must be wrapped in parentheses, and you join them with the bitwise operators & (AND) or | (OR). Using Python's and / or keywords will raise a ValueError because they cannot operate element-wise on Series.

# Rows where age > 30 AND score >= 85
print(df[(df['age'] > 30) & (df['score'] >= 85)])
#     name  age  score
# d  David   40     95

# Rows where age < 28 OR score > 90
print(df[(df['age'] < 28) | (df['score'] > 90)])
#     name  age  score
# a  Alice   25     88
# b    Bob   30     92
# d  David   40     95

The isin() method provides a clean shorthand when you want to match against a set of values, replacing a long chain of | conditions:

# Rows where name is Alice or Eve
print(df[df['name'].isin(['Alice', 'Eve'])])
#     name  age  score
# a  Alice   25     88
# e    Eve   45     84

To negate a condition — selecting rows that do not match — use the tilde ~ operator. It inverts every True to False and vice versa in the boolean mask:

# Rows where name is NOT Alice or Eve
print(df[~df['name'].isin(['Alice', 'Eve'])])
#     name  age  score
# b    Bob   30     92
# c  Carol   35     79
# d  David   40     95

Selecting Single Columns and Rows

Accessing a single column with single-bracket notation — df['col'] — returns a Pandas Series. The Series carries the same index as the DataFrame but represents only one dimension of data. Using double-bracket notation — df[['col']] — wraps the result in a list, which signals to Pandas to return a single-column DataFrame instead. This distinction has real consequences: many DataFrame methods are not available on a Series, so choosing the right return type matters.

print(type(df['age']))        # <class 'pandas.core.series.Series'>
print(type(df[['age']]))      # <class 'pandas.core.frame.DataFrame'>

# Selecting multiple columns always requires double brackets (a list)
print(df[['name', 'score']])
#     name  score
# a  Alice     88
# b    Bob     92
# c  Carol     79
# d  David     95
# e    Eve     84

Dot notation (df.col) is a syntactic shortcut that works for column names that are valid Python identifiers and do not conflict with existing DataFrame attributes or methods. It reads cleanly in exploratory code but should be avoided in production pipelines because it silently fails (returning a method object instead of a column) when the column name collides with a built-in like df.count or contains spaces.

print(df.age)       # Works fine — returns the 'age' Series
# df.score works too, but df.name would return the DataFrame's .name attribute, not the column!

Setting and Resetting the Index

By default, Pandas assigns a RangeIndex (0, 1, 2, …) to a DataFrame. You can replace this with any existing column using set_index(), which turns that column into the row labels. Once set, that column is no longer visible among the regular columns — it becomes the index.

df2 = df.reset_index()   # bring label index back as a column first
df2.columns = ['label', 'name', 'age', 'score']

df_indexed = df2.set_index('name')
print(df_indexed)
#        label  age  score
# name
# Alice      a   25     88
# Bob        b   30     92
# Carol      c   35     79
# David      d   40     95
# Eve        e   45     84

# Now loc with the name is intuitive
print(df_indexed.loc['Carol', 'score'])   # 79

reset_index() reverses this operation. By default it moves the current index back into a column and replaces it with the default integer range. Pass drop=True to discard the old index entirely rather than re-inserting it:

df_reset = df_indexed.reset_index()         # 'name' becomes a column again
df_dropped = df_indexed.reset_index(drop=True)  # 'name' discarded, plain 0,1,2… index restored

A MultiIndex (hierarchical index) is created by passing a list of column names to set_index(). This is useful for data with natural grouping — for example, a dataset indexed by both country and city. MultiIndex DataFrames are accessed with tuples inside loc:

multi_data = {
    'country': ['USA', 'USA', 'UK', 'UK'],
    'city':    ['New York', 'Los Angeles', 'London', 'Manchester'],
    'pop_m':   [8.3, 4.0, 9.0, 2.8]
}
mdf = pd.DataFrame(multi_data).set_index(['country', 'city'])
print(mdf.loc[('USA', 'New York'), 'pop_m'])   # 8.3

Slicing Rows with Basic Bracket Notation

When you pass a slice directly to square brackets — df[0:3] — Pandas interprets it as a row slice by position, following standard Python exclusive-stop rules. This is a quirky exception: normally df[something] selects columns, but a slice object triggers row selection instead.

print(df[1:3])
#     name  age  score
# b    Bob   30     92
# c  Carol   35     79
# (rows at positions 1 and 2; position 3 excluded)

This shorthand is convenient for quick inspection but comes with caveats. It operates on positional indices even when the DataFrame has a custom label-based index — meaning df['b':'d'] works only if the index contains string labels, while df[1:3] always picks the second and third rows by position. For anything beyond simple exploration, prefer loc or iloc for clarity and reliability.

Combining Indexing with Data Manipulation

Indexing is not just for reading data — it is equally important for writing data. Assigning a value to a loc or iloc selection modifies the DataFrame in place:

# Correct way to update a specific cell
df.loc['c', 'score'] = 99
print(df.loc['c', 'score'])   # 99

# Update an entire column subset matching a condition
df.loc[df['age'] >= 40, 'score'] = df.loc[df['age'] >= 40, 'score'] + 5

One of the most dangerous pitfalls in Pandas is chained indexing — accessing data through two consecutive bracket operations, such as df['score'][2]. This first produces a temporary Series, then indexes into it. Pandas may or may not propagate an assignment through the chain, producing the infamous SettingWithCopyWarning and silently failing to update the original DataFrame. Always use a single loc or iloc call for any write operation:

# AVOID — chained indexing, assignment may not work:
df['score'][2] = 50   # unreliable

# CORRECT — single loc call:
df.loc['c', 'score'] = 50   # guaranteed to modify df

When you need a standalone copy of a subset to work with independently — without affecting the original DataFrame — call .copy() explicitly:

subset = df[df['age'] > 30].copy()
subset['score'] = 0    # modifies only subset, not df

Without .copy(), Pandas may create a view rather than a new object depending on the operation. Modifying the view can silently modify the original, or you may receive a warning. Using .copy() removes all ambiguity.

Combining conditional filtering with targeted column assignment is among the most powerful patterns for data cleaning. For example, capping outlier scores, recoding categorical values, or flagging records — all can be done in a single expressive loc statement:

# Recode: anyone with score below 80 gets a 'needs_review' flag
df['flag'] = ''
df.loc[df['score'] < 80, 'flag'] = 'needs_review'
print(df[['name', 'score', 'flag']])
#     name  score          flag
# a  Alice     88
# b    Bob     92
# c  Carol     50  needs_review
# d  David    100
# e    Eve     89

Mastering these indexing tools — understanding not just their syntax but their underlying rules about labels versus positions, inclusive versus exclusive stops, views versus copies, and chained versus atomic assignments — is the foundation for writing correct, efficient, and readable data manipulation code in Pandas.
