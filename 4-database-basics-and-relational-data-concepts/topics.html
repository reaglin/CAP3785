4 - Database Basics and Relational Data Concepts — Module Topics

Introduction to Databases

This topic covers what databases are, why they are used, and how they differ from other data storage methods such as spreadsheets or flat files.
  • What Is a Database? — A database is an organized collection of structured data stored and managed so it can be easily accessed, updated, and retrieved.
  • Why Databases Are Used — Databases solve critical problems around data volume, consistency, and multi-user access that simpler storage methods cannot handle.
  • Databases vs. Spreadsheets — Spreadsheets such as Microsoft Excel are familiar data tools, but they differ fundamentally from databases in purpose, scale, and capability.
  • Databases vs. Flat Files — Flat files, such as CSV or plain-text files, store data as simple sequences of records with no built-in structure or relationships.
  • Types of Databases — There are several categories of databases, each designed for different use cases, with relational databases being the most widely used in business contexts.
  • Key Characteristics of a Well-Designed Database — Effective databases share a set of core characteristics that distinguish them from informal data storage and make them reliable for business use.

Relational Database Concepts

This topic introduces the relational model, explaining how data is organized into structured tables and how this model supports efficient data management.
  • The Relational Model — The relational model is a framework for organizing data into structured tables that relate to one another through shared values.
  • Tables, Rows, and Columns — A table is the fundamental building block of a relational database, organizing data into rows and columns in a grid-like structure.
  • Keys: Primary and Foreign — Keys are special columns used to uniquely identify records and establish relationships between tables.
  • Schemas and Database Structure — A schema is the formal blueprint of a database, defining the tables, columns, data types, and relationships that make up its structure.
  • Relationships Between Tables — Relationships define how tables are connected to each other and are a core feature of the relational model that enables data to be linked across multiple tables.
  • Benefits of the Relational Model for Data Management — The relational model supports efficient data management by reducing redundancy, improving data integrity, and enabling flexible querying.

Tables, Rows, and Columns

This topic explores the fundamental building blocks of relational databases, including how tables are structured with rows representing records and columns representing attributes.
  • What Is a Table? — A table is the primary structure used to organize and store data in a relational database, similar in appearance to a spreadsheet.
  • Rows as Records — Each row in a table represents a single, complete record — one specific instance of the entity the table describes.
  • Columns as Attributes — Each column in a table represents a specific attribute or characteristic shared by every record in that table.
  • The Intersection: Cells — The specific location where a single row meets a single column is called a cell, and it holds one individual data value.
  • Table Structure and Consistency — All rows in a table share the same column structure, enforcing a uniform and predictable format for every record stored.
  • Real-World Analogy: The Spreadsheet Comparison — Relational database tables are often compared to spreadsheet worksheets because both organize data into rows and columns in a grid format.

Keys and Identifiers

This topic explains primary keys, foreign keys, and unique identifiers, and how they are used to uniquely identify records and link data across tables.
  • What Is a Primary Key? — A primary key is a column or set of columns that uniquely identifies each record in a database table.
  • Natural Keys vs. Surrogate Keys — Primary keys can be derived from real-world data (natural keys) or artificially generated by the database system (surrogate keys).
  • What Is a Foreign Key? — A foreign key is a column in one table that references the primary key of another table, establishing a link between the two tables.
  • Referential Integrity — Referential integrity is the rule that ensures foreign key values always correspond to valid primary key values in the related table.
  • Unique Constraints and Unique Identifiers — A unique constraint ensures that all values in a column are distinct, similar to a primary key but without necessarily serving as the table's main identifier.
  • Composite Keys — A composite key uses two or more columns together to uniquely identify a record when no single column is sufficient on its own.
  • How Keys Link Data Across Tables — Keys are the foundation of relational databases, enabling data stored in separate tables to be connected and queried together meaningfully.

Database Schemas

This topic covers what a database schema is, how it defines the structure and organization of a database, and how to interpret a schema diagram.
  • What Is a Database Schema? — A database schema is a formal blueprint that defines the structure and organization of an entire database.
  • Components of a Schema — A schema is made up of several structural components that together describe how data is organized.
  • Schema vs. Database Instance — It is important to distinguish between the schema (structure) and the database instance (the actual data stored at a given moment).
  • Types of Schemas — Database schemas can be described at different levels of abstraction, from the physical storage level to the user-facing view.
  • Reading a Schema Diagram — A schema diagram (also called an Entity-Relationship diagram or ERD) is a visual representation of a database schema, showing tables and how they connect.
  • Why Schemas Matter for Data Analysis — Understanding the schema of a database is a critical first step before performing any data querying or analysis.

Relationships Between Tables

This topic examines how tables relate to one another through one-to-one, one-to-many, and many-to-many relationships, and why these relationships are central to relational databases.
  • What Is a Table Relationship? — A table relationship defines how rows in one table are logically connected to rows in another table within a relational database.
  • One-to-One Relationships — A one-to-one relationship means that each row in Table A corresponds to exactly one row in Table B, and vice versa.
  • One-to-Many Relationships — A one-to-many relationship is the most common type, where a single row in one table can be associated with multiple rows in another table.
  • Many-to-Many Relationships — A many-to-many relationship exists when multiple rows in Table A can relate to multiple rows in Table B, and vice versa.
  • Junction Tables and How They Work — A junction table (also called a linking or associative table) sits between two tables in a many-to-many relationship, storing the combinations of related rows.
  • Why Relationships Are Central to Relational Databases — Relationships are the defining feature of relational databases, enabling data from multiple tables to be combined accurately and efficiently.

Foundations for SQL Querying

This topic connects relational database concepts to practical SQL use, preparing students to query and analyze structured data by understanding how data is stored and linked.
  • From Tables to Queries: Bridging Structure and Retrieval — Understanding how data is organized in tables is the essential first step before writing any SQL query.
  • Primary Keys and Unique Row Identification — A primary key is a column or combination of columns that uniquely identifies each row in a table, and SQL queries rely on this uniqueness to retrieve precise records.
  • Foreign Keys and Querying Across Related Tables — Foreign keys are columns that reference the primary key of another table, creating the links that make relational databases powerful for multi-table queries.
  • Reading a Database Schema Before Writing Queries — A schema is a blueprint of the database that documents all tables, columns, data types, and relationships, and reviewing it is a critical pre-query step.
  • Structured Data and Why SQL Is the Right Tool — Relational databases store structured data — data organized into rows and columns with defined types — which is precisely the format SQL is designed to query.
  • How Relationships Shape Query Logic — The one-to-many and many-to-many relationships defined in a relational database directly determine how you structure JOIN operations in SQL.
